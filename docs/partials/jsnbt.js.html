Upon grunting, the jsnbt.js file is written to the root of the web application (www/jsnbt.js)

This starts a jsnbt js namespace that exposes all merged configuration from all installed modules, as well as ap powerful data api.

<script type="text/javascript" src="jsnbt.js"></script>

The admin and the site have access to the app data through this api.

For each data collection, api functions are generated for CRUD use for these collections as the following

// read jsnbt.db.nodes.get({ id: 'XXX' }, function (err, res) { if (err) { throw err; } else { console.log(res); } }); // create jsnbt.db.nodes.post({ text: '' }, function (err, res) { if (err) { throw err; } else { console.log(res); } }); // update jsnbt.db.nodes.put({ id: 'XXX' }, { text: '' }, function (err, res) { if (err) { throw err; } else { console.log(res); } }); // delete jsnbt.db.nodes.del({ id: 'XXX' }, function (err, res) { if (err) { throw err; } else { console.log(res); } });
