server

The server object created on application start.

This singleton object gives you access to the app and jsnbt configurations and other singleton mechanisms as cache and messaging engines.

app object
the app instance that the server deployd on.

path string
the path from which the app is running
dbg bool
the debug status of the app running
modules string
the modules loaded into the jsnbt engine
init(options, module) function
initiates the application
createServer(options) function
creates the dpd server for the application
jsnbt object
returns the running jsnbt configuration.

languages array of strings
the available languages
countries array of strings
the available countries
modules array of objects
the loaded modules
configs object
the configurations injected by any modules per domain
localization bool
whether the application runs on multiple languages
this value is usually set by the site module alone
locale string
the singleton language code (applicable only when the "localization" property is set to true)
this value is usually set by the site module alone
ssl bool
whether the site or part of it is under ssl or not
this value is usually set by the site module alone
restricted bool
whether the site or part of it is restricted to specific user roles
this value is usually set by the site module alone
jsModules array of strings
the injected angular module names from installed modules
scripts array of strings
extra head scripts (for the admin pages)
fileGroups array of strings
the available file group names.
images array of objects
the available image transformations.
entities array of objects
the entities.
roles array of objects
the user roles.
sections array of objects
the sections with their available roles.
collections object
the data collections along with their restrictions, events and schema validations.
lists array of objects
the data lists
layouts array of objects
the layouts.
containers array of objects
the containers.
templates array of objects
the templates.
routes array of objects
the routes.
messager object
the messager engine. it serves the below for mail and sms

getTemplate(templateCode, callback) function
retrieves a template and returns it as an argument on the callback function
getModel(templateCode, callback) function
retrieves the debug model for a template code and returns it as an argument on the callback function
parseTemplate(template, model, callback) function
parses a template with a model and returns the parsed result as an argument on the callback function
getSender(dpd, callback) function
creates a sender and returns it as an argument on the callback function
cache object
the server cache
getPath(path) function
returns the server path of a relative path
start(next) function
starts the server
