You might not have a config file, and in the index file's getConfig() method just return there the json configuration object.
This is just to separate the config from the functionality!

domain string
the domain identifier for the module. this should be easy to identify, and if possible limited to one word for consistency
browsable bool
indicates whether this module is to be shown in the admin modules list
pointed bool
indicates whether if the content of this module can be linked to a pointer node
version string
returns the module version 
version: require('../../package.json').version
jsModule string
the js angular module to be injected in the admin angular module configuration
entities array of objects
entity type injections for the data in the "nodes" collection 
entities: [{
    name: 'item'
}, {
    name: 'folder',
    allowed: ['folder', 'item']
}]
roles array of objects
custom roles that might be needed by the module 
roles: [{
    name: 'module-role'
}, {
    name: 'module-super-role',
    inherits: ['module-role']
}]
sections array of objects
custom sections that are used by the module's pages, along with their role access lists 
sections: [{
    name: 'module-section',
    roles: ['module-role']
}, {
    name: 'module-super-section',
    roles: ['module-super-role']
}]
permissions array of objects
custom permissions for the dpd resources 
permissions: [{
    collection: 'custom-collection',
    roles: [{
        role: 'module-role',
        crud: ['R']
    }, {
        role: 'module-super-role',
        crud: ['C, 'R', 'U', 'D']
    }]
}]
fileGroups array of strings
file groups for the file manager 
fileGroups: ['users', 'stores']
images array of objects
custom image type cropping and sizing specifications 
images: [{
    name: 'thumb',
    processors: [{
        type: 'crop',
        options: {
            x: 0,
            y: 0,
            width: 60,
            height: 60
        }
    }, {
        type: 'fit',
        options: {
            width: 60,
            height: 60
        }
    }, {
        type: 'stretch',
        options: {
            width: 60,
            height: 60
        }
    }]
}]
scripts array of strings
a collection of js script urls to be included in the admin pages 
scripts: [
    'http://maps.google.com/maps/api/js?sensor=true'
]
lists array of objects
custom data lists along with their role permissions 
lists: [{
    id: 'sample1',
    name: 'Sample 01',
    form: 'tmpl/customModule/sample1.html',
    localized: true,
    permissions: [{
        role: 'module-role',
        crud: ['R']
    }, {
        role: 'module-super-role',
        crud: ['C', 'R', 'U', 'D']
    }]
}]
