domain string
the domain identifier for the module
this should be set to "public" for the site module
public bool
indicates whether if this module is the site module or not
this should be set to "true" for the site module
restricted bool
indicates whether if the site utilizes members or not
if so, there will be permission controls in each node, and if access is denied the user will be redirected to a login page
ssl bool
indicates whether if the site is under ssl or not
locale string (optional)
indicates whether if the site is localized in more than one languages or not
if the locale is specified, the jsnbt will assume that there is one and only content language
injects object
defines the custom content injections to the admin predefined holders 
injects: {
    navigation: [{
        index: 5,
        identifier: 'extra',
        name: 'extra',
        url: '/extra',
    }],
    dashboard: 'tmpl/public/injects/dashboard.html',
    content: 'tmpl/public/injects/content.html',
    settings: 'tmpl/public/injects/settings.html'
}
layouts array of objects
custom layouts of the site
the layouts defined here, will be available to be bound to nodes and provide administrative layout management for their front end templates 
layouts: [{
    id: 'global',
    name: 'global layout',
    form: 'tmpl/public/layouts/global.html'
}, {
    id: 'eshop',
    name: 'eshop layout',
    form: 'tmpl/public/layouts/eshop.html'
}]
containers array of objects
the available containers of the site
these containers are available to the container controls 
containers: [{
    id: 'sampleContainer1',
    name: 'sample container A',
    html: '/tmpl/partial/containers/sampleContainerA.html',
}, {
    id: 'sampleContainer2',
    name: 'sample container B',
    html: '/tmpl/partial/containers/sampleContainerB.html'
}]
templates array of objects
the available templates of the site
these template are available to the template dropdown control on every node
templates are assosiated with "form" guidance html parts, that are used by the node to save additional data that concern this only template 
templates: [{
    id: 'text',
    name: 'text page',
    html: '/tmpl/text.html',
    form: 'tmpl/public/forms/page/text.html',
    restricted: ['page']
}]
lists array of objects
data lists along with their role permissions
the content of these lists is managed in the data section of the admin 
lists: [{
    id: 'sample1',
    name: 'Sample 01',
    form: 'tmpl/customModule/sample1.html',
    localized: true
}]
routes array of objects
custom routes implemented by the public module
a route must define an id and a display name, as well as a function ("fn") name that exists in the public module's index file. 
routes: [{
    id: 'search',
    name: 'Search results',
    fn: 'routeSearch'
}]
