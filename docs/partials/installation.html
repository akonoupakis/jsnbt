First, download a copy of jsnbt-strap-angular from https://github.com/akonoupakis/jsnbt-strap-angular.
This is a quick bootstrap to strart you up. Put the files in a folder of your choice, and lets have a look at a few of them.


  • package.json

    The file should have filled the grunt dependencies that will be needed.
    Also, the "main" attribute that points to the module's index file!

  • hosts.js

    This is the hosting configuration file. This should be kept out of source control systems and should not be overriden during deployment.

  • gulpfile.js

    This is used by the gulp task runner to bundle the site to the deployment target folder (/www) in order to be started by the node web server.
    The jsnbt gulp process is fairly complicated, so it has been embedded in the jsnbt core code and will be covered in next chapters.
    In your gulpfile, you should point the gulp function to jsnbt/gulpfile.js.

  • src/app/index.js

    This is the entry point of your site to the cms engine. We will go through the module index file in next chapters.

  • src/cfg/config.js

    This is the configuration file of your site.

  • src/web/bower.json

    Specify here the bower dependencies that your site will need.
    The "deploy" section of this config file, guides the system to copy the downloaded files to a target location (we will see later, that this target location is the lib folder within our app).

  • server-app.js

    Runs the application in node.

install

Point to the app folder and:
Run npm install to install all dependencies.
Run npm install jsnbt --save to install jsnbt.

run

Run gulp dev or gulp prod to structure-up the app (this also downloads any bower components needed).
Run node server-app.js to run the app.

you're done

You can hit the /admin section to create your first sa user and access the administration panel.
